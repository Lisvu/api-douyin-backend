## 开发与验收检查清单

可以直接使用以下清单在开发过程中进行对照打勾。

### 1. 前端联调（5 分）

- [ ] F01：实现了所有后端接口的对接，流程能跑通，不纠结 UI 样式。

### 2. 核心业务 API（40 分）

- [ ] F08：用户 API：注册、登录、注销功能可用。
- [ ] F02：视频流 API：返回点赞数最高的视频。
- [ ] F02：去重逻辑：记录用户的观看历史，确保推荐流中不出现已看视频。
- [ ] F03：切换 API：能够根据当前视频状态加载上一个 / 下一个视频。
- [ ] F04：点赞 API：点赞功能正常（注意防并发或重复点赞逻辑）。
- [ ] F14：消息通知 API：可查询互动通知列表，未读角标与已读标记正常。
- [x] F15：评论 API：可分页查看评论、发表评论，评论数正确回显。
- [x] F16：用户主页 API：可查看用户资料及发布/点赞视频分页列表；前端点头像可进入主页。
- [ ] F05：发布 API：视频上传与发布成功。
- [ ] F06：列表 API：请求“我的视频”时，传入 `page` 和 `size` 参数并正确返回分页数据。
- [ ] F07：删除 API：删除时校验当前登录用户是否为视频所有者。

### 3. 底层与中间件（25 分）

- [ ] F09：DB 验收：数据库表结构（用户表、视频表、点赞记录表、观看历史表等）设计文档或 SQL。
- [ ] F10：存储验收：明确视频文件存放路径及访问 URL 生成逻辑。
- [ ] F11：日志拦截器 / 中间件：统一拦截请求，打印或记录 Request Payload 和 Response Body。
- [ ] F12：耗时统计器：在中间件记录 Request Start Time 和 End Time，输出接口执行耗时。
- [ ] F13：全局鉴权网关 / 拦截器：统一校验 Token 或 Session，确保未登录用户无法访问核心数据，越权操作会被拦截。
