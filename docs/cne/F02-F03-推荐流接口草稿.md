## 功能对应关系

| 编号 | 功能 | 接口 |
| --- | --- | --- |
| F02 | 按点赞数倒序推荐未看视频 | `GET /api/v1/videos/recommendations` |
| F02 | 观看记录（去重依据） | `POST /api/v1/videos/{id}/views` |
| F02 | 重置观看记录（调试） | `DELETE /api/v1/users/me/views` |
| F03 | 视频切换 | 前端本地索引切换 + 播放成功时调用观看记录接口 |

**与组员 B（F04）的字段约定**：推荐列表一并返回 `likes_count`、`is_liked`；点赞操作使用 `PUT /api/v1/videos/{id}/like`，前端在点赞成功后更新本地 `videos` 状态。

## 1. 获取推荐视频列表

### `GET /api/v1/videos/recommendations`

| 项 | 说明 |
| --- | --- |
| 认证 | 是，`Authorization: Bearer <token>` |
| 业务规则 | 排除当前用户已在 `views` 表中的视频；按 `likes_count` **降序** |
| 成功状态码 | `200 OK` |
| 失败状态码 | `401 Unauthorized`（未登录 / Token 无效）、`500 Internal Server Error` |

#### 请求

无请求体，无必填 Query 参数。

#### 响应字段

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `success` | boolean | 是 | 是否成功 |
| `videos` | array | 是 | 推荐视频列表，可为空数组 |
| `allViewed` | boolean | 是 | 当 `videos` 为空时为 `true`，表示当前用户已看完所有可推荐视频 |
| `totalCount` | number | 是 | 数据库中视频总数（**含**已观看） |
| `message` | string | 否 | 失败时错误说明 |

#### `videos[]` 单项字段

| 字段 | 类型 | 说明 | 前端展示 |
| --- | --- | --- | --- |
| `id` | number | 视频主键 | 播放、上报观看 |
| `user_id` | number | 发布者用户 ID | 扩展用 |
| `title` | string | 标题 | F01 播放区标题 |
| `description` | string | 描述 | 可选副文案 |
| `video_url` | string | 视频文件 URL（可为绝对路径或相对 `/uploads/...`） | `<video src>` |
| `cover_url` | string | 封面 URL | 封面图 |
| `likes_count` | number | 点赞总数（兼容字段，同 `likeCount`） | F01 / F04 展示 |
| `likeCount` | number | **规范字段**：点赞总数 | F01 / F04 展示 |
| `liked` | boolean | **规范字段**：当前用户是否已赞 | F04 点赞按钮 |
| `is_liked` | `0` \| `1` | 兼容字段 | F04 |
| `created_at` | string (ISO-8601) | 创建时间 | 可选 |
| `creator_name` | string | 发布者用户名 | F01 作者展示 |

#### 成功响应示例

```json
{
  "success": true,
  "videos": [
    {
      "id": 2,
      "user_id": 1,
      "title": "DCloud 移动端跨平台技术",
      "description": "...",
      "video_url": "https://...",
      "cover_url": "https://...",
      "likes_count": 580,
      "created_at": "2026-01-01T10:00:00",
      "creator_name": "douyin_creator",
      "is_liked": 0
    }
  ],
  "allViewed": false,
  "totalCount": 6
}
```

#### 全部看完示例

```json
{
  "success": true,
  "videos": [],
  "allViewed": true,
  "totalCount": 6
}
```

#### 实现位置

- Controller：`VideoController#getRecommendations`
- 查询：`VideoRepository#findRecommendedVideosForUser`

## 2. 标记视频已观看

### `POST /api/v1/videos/{id}/views`

| 项 | 说明 |
| --- | --- |
| 认证 | 是 |
| 路径参数 | `id`：视频 ID（Long） |
| 幂等性 | 同一用户对同一视频重复调用不重复插入（应用层 `exists` + 数据库 `UNIQUE(user_id, video_id)`） |
| 成功状态码 | `200 OK` |

#### 请求

无请求体。

#### 响应字段

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `success` | boolean | 是否成功 |
| `message` | string | 如 `"Video marked as viewed."` |

#### 成功响应示例

```json
{
  "success": true,
  "message": "Video marked as viewed."
}
```

#### 实现位置

- Controller：`VideoController#recordView`
- 实体：`View`（`views` 表）

## 3. 重置当前用户观看记录

### `DELETE /api/v1/users/me/views`

| 项 | 说明 |
| --- | --- |
| 认证 | 是 |
| 语义 | 删除当前登录用户在 `views` 表中的全部记录 |
| 成功状态码 | `200 OK` |

#### 响应示例

```json
{
  "success": true,
  "message": "Your watch history has been reset. All videos can be recommended again!"
}
```

#### 实现位置

- Controller：`VideoController#resetViews`
- Repository：`ViewRepository#deleteByUserId`

## 4. F03 切换相关说明（非独立 REST 资源）

本项目 **不** 提供 `GET /feed/next` 或 `GET /feed/prev`。F03 交互约定如下：

| 步骤 | 行为 | 触发的接口 |
| --- | --- | --- |
| 登录 / 刷新推荐 | 拉取完整未看列表 | `GET /api/v1/videos/recommendations` |
| 上滑 / 下滑 / 箭头切换 | 前端 `currentIndex ± 1`，切换 `<video>` 片源 | 无 GET（本地切换） |
| 新视频开始播放成功 | 写入观看记录 | `POST /api/v1/videos/{id}/views` |
| 点击「重置观看记录」 | 清空历史并重新拉列表 | `DELETE /api/v1/users/me/views` → `GET .../recommendations` |

详细时序与边界见：`docs/F02-F03-观看记录与切换方案.md`。

## 6. 自测要点（验收）

| 检查项 | 预期 |
| --- | --- |
| 未观看任何视频时 `GET recommendations` | 返回种子数据 6 条，`likes_count` 降序 |
| 对某视频 `POST views` 后再次 `GET` | 该 `id` 不再出现在 `videos` |
| `DELETE users/me/views` 后 `GET` | 恢复完整推荐列表 |
| 前端切换视频 | 监控面板可见 `POST /videos/{id}/views` |

### PowerShell 快速验证

```powershell
$login = Invoke-RestMethod -Uri "http://localhost:8080/api/v1/auth/login" -Method POST -ContentType "application/json" -Body '{"username":"douyin_creator","password":"password123"}'
$h = @{ Authorization = "Bearer $($login.token)" }

Invoke-RestMethod -Uri "http://localhost:8080/api/v1/videos/recommendations" -Headers $h |
  Select-Object success, allViewed, totalCount, @{n='count';e={$_.videos.Count}}

Invoke-RestMethod -Uri "http://localhost:8080/api/v1/videos/2/views" -Method POST -Headers $h
Invoke-RestMethod -Uri "http://localhost:8080/api/v1/videos/recommendations" -Headers $h
```

默认测试账号：`douyin_creator` / `password123`（见 `ApiDouyinApplication` 种子数据）。
